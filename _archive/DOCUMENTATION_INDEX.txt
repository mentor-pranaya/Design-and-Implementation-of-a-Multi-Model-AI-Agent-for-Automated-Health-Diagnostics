════════════════════════════════════════════════════════════════════════════
                         TROUBLESHOOTING GUIDE
════════════════════════════════════════════════════════════════════════════

Problem: Site can't be reached
→ Read: COMPLETE_SERVER_FIX_GUIDE.md
→ Run: diagnose.bat
→ Solution: Usually Python not installed

Problem: "Python not recognized"
→ Read: FIX_SERVER_STEP_BY_STEP.txt
→ Solution: Restart computer after Python install

Problem: "Address already in use"
→ Solution: Kill process using port 10000, try again

Problem: Server closes immediately
→ Run: diagnose.bat (shows detailed error)

Problem: Dependencies not installing
→ Check: Do you have internet connection?
→ Try: python -m pip install --upgrade pip
→ Then run: setup_and_start.bat again

Response Caching:
  • First request: full analysis (may take 5-30 seconds)
  • Repeated requests: instant results from cache (50-100ms)
  • Cache timeout: 1 hour
  • Cache hit indicator: Check /api/status

Parallel Processing:
  • Multiple analysis agents run simultaneously
  • 4 parallel workers for concurrent tasks
  • Reduces total analysis time significantly

Connection Pooling:
  • Database connections reused efficiently
  • 10 connections in pool
  • Reduces connection overhead

Compression:
  • GZIP compression on all responses
  • Reduces bandwidth by 50-80%
  • Transparent to client

Monitoring:
  • Real-time performance metrics
  • Cache hit/miss statistics
  • Average response times
  • Throughput monitoring
