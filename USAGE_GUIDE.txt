                        WHAT EACH PART MEANS
                        
    ✓ summary
      → Clinical overview of abnormalities and risks
      
    ✓ key_findings
      → List of abnormal lab parameters with explanations
      
    ✓ risks
      → Domain-level risk assessment (cardiac, diabetes, etc.)
        with severity scores and confidence
        
    ✓ recommendations
      → Personalized actionable advice based on:
        - Detected abnormalities
        - Identified risks
        - User context (age, gender, lifestyle)
        
    ✓ urgency_level
      → "low", "moderate", or "high"
        Determines response priority
        
    ✓ confidence_summary
      → Quick confidence metric for top risk
      
    ✓ disclaimer
      → Legal notice (required for medical reports)

═══════════════════════════════════════════════════════════════════════════════

                        USING WITH PYTHON
                        
    import requests
    
    with open("report.pdf", "rb") as f:
        response = requests.post(
            "http://127.0.0.1:8000/analyze",
            files={"file": f},
            data={
                "age": 45,
                "gender": "Male",
                "medical_history": "Hypertension"
            }
        )
    
    report = response.json()
    print(f"Status: {report['status']}")
    print(f"Summary: {report['data']['summary']}")
    print(f"Urgency: {report['data']['urgency_level']}")

                        TROUBLESHOOTING
                        
    Port in use?
    → Kill process: taskkill /PID <PID> /F
    → Or use different port: uvicorn api.main:app --port 9000
    
    Can't upload file?
    → Check format (must be PDF, JPG, PNG, or JSON)
    → Check file size
    → Try in Swagger UI first (http://127.0.0.1:8000/docs)
    
    Slow response?
    → First request is slow (OCR model loads)
    → Subsequent requests are fast
    
    Tests fail?
    → Run: pytest tests/ -v
    → Check Python version: python --version (need 3.8+)
    → Reinstall deps: pip install -r requirements.txt (if exists)
